let和const

  1. let和const没有变量提升
  2. let定义后可以进行修改
  3. const引用对象定义后指针指向可以修改，但基本类型不能修改
  4. let、const不能重复声明

箭头函数

  1. 没有自己的this
  2. 不能作为构造函数，不可以new
  3. 没有arguments对象
  4. 没有原型对象

  3. 与解构赋值结合使用

数值扩展

  1. Number.EPSILON 是JavaScript中表示的最小精度
  2. Number.isFinite 检测一个数值是否是有限数
  3. Number.isNaN 检测一个数值是否是NaN，Number.isNaN()只有对于NaN才返回true，非NaN一律返回false。
  4. Number.parseInt Number.parseFloat 字符串转化为整数，es6将这两个方法移到Number，逐步减少全局性方法，使语言逐步模块化。
  5. Number.isInteger 判断一个数是否为整数，与传统的全局方法isFinite()的区别在于，传统方法先调用Number()将非数值的值转为数值，再进行判断， 而这两个新方法只对数值有效，Number.isFinite()对于非数值一律返回false,
  6. Math.trunc 将数字的小数部分抹掉
  7. Math.sign 判断一个数是整数还是负数还是0
  8. 数值分隔符
    • ES2021，允许 JavaScript 的数值使用下划线（_）作为分隔符。
    • 这个数值分隔符没有指定间隔的位数，也就是说，可以每三位添加一个分隔符，也可以每一位、每两位、每四位添加一个。
    • 小数和科学计数法也可以使用数值分隔符。
    • 不能放在数值的最前面（leading）或最后面（trailing）。
    • 不能两个或两个以上的分隔符连在一起。
    • 小数点的前后不能有分隔符。
    • 科学计数法里面，表示指数的e或E前后不能有分隔符。
    • 数值分隔符只是一种书写便利，对于 JavaScript 内部数值的存储和输出，并没有影响。
    • 将字符串转成数值的函数，不支持数值分隔符。

对象方法扩展

  1. Object.is 判断两个值是否相等
  2. Object.assign 对象的合并
  3. Object.setPrototypeOf 设置原型对象
  4. Object.keys()、Object.values()、Object.entries()
  5. Object.getOwnPropertyDescriptors() 对象属性的描述
  6. Object.getOwnPropertyNames(obj) 返回一个数组，含对象自身的所有属性（不含 Symbol 属性，但是包括不可枚举属性）的键名。
  7. Object.getOwnPropertySymbols(obj) Object.getOwnPropertySymbols返回一个数组，包含对象自身的所有 Symbol 属性的键名。

字符串方法扩展

  1. includes(‘字符串’,’从第N个字符开始’)：返回布尔值，表示是否找到了参数字符串。
  2. startsWith(‘字符串’,’从第N个字符开始’)：返回布尔值，表示参数字符串是否在原字符串的头部。
  3. endsWith(‘字符串’,’前N个字符’)：返回布尔值，表示参数字符串是否在原字符串的尾部。
  4. repeat(重复次数)
    • repeat方法返回一个新字符串，表示将原字符串重复n次。
    • 参数如果是小数，会被取整。
    • 如果repeat的参数是负数或者Infinity，会报错。
    • 参数NaN等同于 0。
    • 如果repeat的参数是字符串，则会先转换成数字。
  5. padStart(位数,’补全字符串’)、padEnd(位数,’补全字符串’)
    • padStart()用于头部补全，padEnd()用于尾部补全。
    • 如果原字符串的长度，等于或大于最大长度，则字符串补全不生效，返回原字符串
    • 如果用来补全的字符串与原字符串，两者的长度之和超过了最大长度，则会截去超出位数的补全字符串。
    • 如果省略第二个参数，默认使用空格补全长度。
  6. at(指定位置) 返回参数指定位置的字符，支持负索引
