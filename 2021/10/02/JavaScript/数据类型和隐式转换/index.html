undefined和null的区别

  1. undefined是未定义，null是定义完没有值
  2. typeof undefined是undefined，typeof nyll是0
  3. 转为数字时undefined是NaN，null是0
  4. null == undefined ，null !== undefined

symbol

属性
  1. 表示独一无二的值
  2. symbol不是构造函数，不能使用new关键字
  3. symbol() 里面的内容是symbol的描述，只是为了方便开发中辨识，并不能代表symbol的值
  4. 每一个symbol类型数据都是独一无二，不能划等号
  5. symbol不能和其他值参与运算
  6. symbol定义的属性无法被Object.keys()或者for…in 遍历出来
  7. symbol类型的属性不能被JSON.stringify()序列化

堆和栈

堆

  1. 引用数据类型存放地点
  2. 占用空间大，大小不稳定
  3. 指针指向堆中的地址
  4. 优先队列

栈

  1. 原始数据类型的存贮地点
  2. 引用数据类型指针存放地点
  3. 占用空间小，大小固定，属于被频繁使用的数据
  4. 先进后出
  5. 由编译器自动分配释放

隐式转换

Boolean判断为false的值

  1. ‘’
  2. null
  3. undefined
  4. 0
  5. false
  6. NaN

==的转化规则

  1. 一边为null，一边为undefined，则为false
  2. 一边为数字，一边为字符串，先将字符串转化为数字再进行比较
  3. 一边为true，则先转化为1再比较，其中一个为false，先转化为0再比较
  4. 一个值为对象，老另一个值为数字或者字符串，将对象转化为原始值在比较

对象转化为原始类型

  1. 会调用ToPrimitive方法
  2. ToPrimitive内部会调用toString和valueOf方法
  3. 目标对象为字符串先调用toString方法，其余先调用valueOf方法

+的转化规则

  1. 其中一个为字符串，或者其中一个为对象，但取得的元素值为字符串，则视为字符串连接符
  2. 两边都是number类型，或者两边都是出来string以外的基本类型，则视为加号
