egg.js基本使用

四月 19, 2022

控制器（controller）

  1. 直接响应数据或渲染模板
  2. 接受用户的输入
  3. 与路由建立对应关系
  4. this.ctx可以获取到当前请求的上下文对象，通过此对象可以便捷的获取到请求与响应的属性与方法

路由

通过路由传参

  1. 获取query参数
  1. 获取params参数
  1. post请求提交数据 post请求刷新页面相当于再次提交表单
  2. 使用restful接口

模板引擎

  1. npm install --save egg-view-nunjucks
  2. 在plugin.js文件中引入插件，在config.default,js中配置插件
  1. 在view中创建模板文件，并在控制器中使用render方法渲染模板

跨域

  1. 使用egg-cors npm install --save egg-cors
  2. 在plugin.js文件中引入插件
  1. 在config.default.js文件中配置egg-cors插件

用户登录状态

  1. 使用cookie与session识别用户
  2. 使用JWT(json web token)识别用户

JWT

生成token

  1. 安装egg-jwt插件：npm install --save egg-jwt
  2. 在plugin.js中引入插件
  1. 配置config.default.js文件，设置secret；注意，secret不能泄露

验证过程

  1. 根据用户信息，生成token，并响应给客户端
  1. 客户端存储在localstorage中
  2. 客户端每次请求数据，请求头携带token
  1. 服务器接收请求，验证请求头的token，验证成功则响应数据

实际项目中token请求应当是封装好的，每次调用接口自动携带

中间件

  1. 在app目录下创建middleware目录
  2. 在middleware中创建js文件
  1. 中间件的使用
    • 写在router文件中，要使用的接口的接口名称和controller中间

数据持久化

sequelize

  1. 下载egg-sequelize： npm install –save egg-sequelize mysql2

  2. 在plugin.js中引入插件

  3. 在config.default.js文件中配置数据库连接

  4. 在app/model文件中创建数据模型

  5. 添加app.js文件，初始化数据库（app.js文件在最外层）

数据操作

在controller中实现数据的增删改查，在真实项目中，controller和操作数据的逻辑要分离，以便于项目的维护和扩展

  1. this.app.model.Clazz.findAll() // 查询数据
  2. this.app.model.Clazz.findAll({where:{id:1}}) // 通过where设置条件查询数据
  3. this.app.model.Clazz.create({name: “xx”}) // 添加数据
  4. this.app.model.Clazz.update({name: “xx”},{where:{id:1}}) // 通过条件修改数据
  5. this.app.model.Clazz.destory({where:{id:1}}) // 通过条件删除数据
  6. 添加外键

service

service就是在复杂业务场景下用于做业务逻辑封装的一个抽象层，提供这个抽象层，有以下优点：

  1. 可以保持controller中逻辑的简洁，controller只起接受数据和返回数据处理的作用
  2. 保持业务逻辑的独立性，抽象出来的service可以被多个controller重复调用
