mini-webpack

处理文件

  1. 对于webpack来说，首先要做的是处理文件信息，获取文件的内容
    • 利用fs.readFileSync读取文件
  2. 根据获取到的文件来获取文件的依赖关系
    • 利用babel/parser将获取的文件转化为AST,并指定sourceType: 'module'使文件可以解析import导入的文件
    • 利用babel/traverse实现对AST的查找，找到其中的node.source.value，即当前文件引用了哪个文件，则认为存在依赖关系
  3. 创建一个deps容器用于收集依赖关系

根据依赖关系生成图结构

  1. 由于文件可能有相互引用的关系，所以文件之间不是简单的树结构，而是图结构
  2. 指定解析的入口文件，一般来说是main.js文件
  3. 将main.js作为入口文件放入一个栈中，遍历栈中元素，再遍历每一个元素的deps属性（即依赖关系），将每一个属性添加上相对路径后添加到栈中，直到栈为空

利用ejs模板文件生成最终打包好的文件

  1. 首先引入提前创建好的模板文件bundle.ejs
  2. 遍历上面生成的图，遍历取得所需要的数据
  3. 指定出口文件./dist/bundle.js
  4. 将取得的数据传给模板文件，利用ejs根据文件生成最后打包生成的文件
  5. 执行build方法，即可以获取打包后的文件

其他文件

bundle.ejs

  1. 为了防止命名冲突，采用id代替name
  2. 生成图结构的时候，把文件的id作为mapping中相对路径的值存储起来

axios

axios是什么

  1. axios 是一个轻量的 HTTP客户端
  2. 特点
    • 从浏览器创建XMLHttpRequests
    • 从node.js创建http请求
    • 支持promise API
    • 拦截请求和响应
    • 转化请求数据和响应数据
    • 取消请求
    • 自动转化JSON数据
    • 客户端支持防御XSRF
  3. 简单请求

为什么要进行封装

  1. axios的API很友好，完全可以在项目中直接使用
  2. 不过随着项目规模的增大，如果没发起一次HTTP请求，就要这些比如设置超时时间、设置请求头、根据项目环境判断使用哪个请求地址、错误处理等重新写一遍，不仅浪费人力，而且会使代码冗余
    比如

如何封装

  1. 约定好请求头、状态码、请求超时时间等
  2. 设置接口请求前缀：根据开发、测试、生产环境的不同，前缀需要加以区分
  3. 请求头：来实现一些具体的业务，必须携带一些参数才可以请求
  4. 状态码：根据接口返回的不同status，来执行不同的业务
  5. 请求方法：根据get、post等方法进行一个再次封装，使用起来更加方便
  6. 请求拦截器：根据请求的请求头设定，来决定哪些请求可以访问
  7. 响应拦截器：根据后端返回来的状态码判定执行不同的业务

nextTick

  2. 定义一个异步方法，采用降级处理的方法，依次判断，执行flushCallbacks方法
    • Promise
    • MutationObserver 监听dom变化

渲染更新原理

由初次渲染原理可知，实例挂载之后，会调用vm._update(vm._render())来更新视图，_update方法内部调用了patch方法，但是不可能每次数据变化都要求用户自己去调用渲染方法更新视图，所以需要一个机制再数据变动的时候自动更新

watcher

首先，我们需要使用到观察者模式来订阅数据的变动，所以新建一个watcher作为观察者，用它来订阅数据的变化，当数据变动之后，通知他去执行某些方法

定义watcher

此外，在组件挂载，即mountComponent中，要定义一个渲染watcher主要功能就是执行核心渲染页面的方法

dep

  1. 随着watcher的增多，需要一个容器来管理，所以产生了dep
  2. dep也是一个构造函数，可以把它理解为观察者模式里面的被观察者
  3. 在subs里面收集watcher，当数据变动的时候通知吱声subs所有的watcher更新
  4. dep.target是一个全局watcher指向，初始状态为null

对象的依赖收集

对象的依赖收集借助于响应式原理，即数据在被访问的时候，把我们定义好的渲染watcher放到dep的subs数组里面，同时把deo实例对象也放到渲染watcher里面取，数据更新的时候就可以通知dep的subs存储的watcher更新

数组的依赖收集

  1. 如果对象属性的值是一个数组，那么执行childOb.dep.depend()收集数组的依赖，如果数组里面还包含数组，需要递归遍历收集
  2. 因为只有访问数据触发了get才会去收集依赖，一开始只是递归对数据进行响应式处理无法收集依赖
  3. depend方法中，如果当前存在watcher，就会触发Dep.target.addDep(this)，把自身dep放到watcher里面，addSub中执行this.subs.push(watcher)

  3. 执行notify方法，触发sub里面watcher的更新方法watcher.update()

初次渲染原理

组件挂载入口

  2. 这个方法放在源码的 lifecycle 文件里面 代表了与生命周期相关 因为我们组件初始渲染前后对应有 beforeMount 和 mounted 生命周期钩子

组件挂载

使用_update方法进行实例挂载，传入_update的是虚拟dom，在_render()进行render函数的虚拟化

组件虚拟化

  1. 在vue的原型上定义了一个_render方法，获取到模板编译生成的render方法，即创建vue实例时传入的render函数或者经过编译，处理el或者template生成的render函数
  2. 同时在vue的原型上定义_c、_v、_s方法，用来创建虚拟DOM元素，虚拟DOM文本和处理模板中的对象 新建vnode类，定义reateElement 和 createTextNode 方法最后都返回 vnode

虚拟DOM转化为真实DOM

  1. 主要方法是patch方法，用来渲染和更新视图 初始渲染和后续更新都是共用这一个方法 只是传入的第一个参数不同 初始渲染总体思路就是根据虚拟 dom(vnode) 调用原生 js 方法创建真实 dom 节点并替换掉 el 选项的位置

动态规划

动态规划问题主要分为三类：0-1背包、完全背包、多重背包

0-1背包

  1. 概念
    • 有n件物品（每件物品只能用一次）和一个最多能背重量为w的背包。第i件物品的重量是weight[i],价值是value[i],每件物品只能用一次，求解将哪些物品装入背包中物品价值总和最大？
  2. 实例
    重量 价值
    物品0 1 15
    物品1 3 20
    物品2 4 30
    背包最大重量为4。问背包能背的物品最大价值是多少？
  3. dp
    • dp[i][j]表示从下标为0-i的物品中选取不超过容量j的最大价值总和
    • dp[i][j]不放物品：dp[i-1][j]表示背包容量为j，里面不放物品i的最大价值。
    • dp[i][j]放物品：dp[i][j]=dp[i-1][j-weight[i]]+valuei。其中，dp[i-1][j-weight[i]]表示减去物品i的重量(j-weight[i])背包能得到的最大价值。
    • 之后再推导出dp[i][j]的最大值，是放入物品i还是不放入物品i
  4. dp数组初始化
  5. 遍历顺序

完全背包问题

  1. 概念
    • 完全背包问题描述：有编号分别为a,b,c,d的四件物品，它们的重量分别是2,3,4,7，它们的价值分别是1,3,5,9，每件物品数量无限个，现在给你个承重为10的背包，如何让背包里装入的物品具有最大的价值总和
    • 完全背包问题与01背包问题的区别在于每一件物品的数量都有无限个，而01背包每件物品数量只有一个

多重背包问题

  1. 概念
    • 多重背包问题描述：有编号分别为a,b,c的三件物品，它们的重量分别是1，2，2，它们的价值分别是6，10，20，他们的数目分别是10，5，2，现在给你个承重为 8 的背包，如何让背包里装入的物品具有最大的价值总和
    • 多重背包和01背包、完全背包的区别：多重背包中每个物品的个数都是给定的，可能不是一个，绝对不是无限个

模板编译原理

vue实例化

  1. vue实例化有两种方式，首先会判断是否有el配置项，如果没有则去判断$mount
  2. 接下来判断是否有template选项，如果有的话解析template到render函数中，如果没有的话，编译el对应的文件作为template
  3. 传入的el或者template选项最后都会被解析成render函数，这样才能保证模板解析的一致性

    模板编译入口

  4. 在上一篇文章，数据初始化之后，就要根据el属性进行模板渲染
  5. 为了对模板进行统一处理，会先判断有没有el属性
  6. 给vue的原型上绑定一个$mount属性，里面包含模板的编译过程
  7. 首先会根据vue实例化的options是否含有render，如果没有render函数，就先要把模板转化为render函数
  8. 依次判断options中的template和el,找到统一的模板后进行转换

    • 优化静态节点
    • 根据ast生成render函数
